Part 2: Model Context Protocol (MCP)

What is MCP?

üîå Definition

Model Context Protocol (MCP) is an open protocol that enables AI applications to securely connect to external data sources and tools. It standardizes how AI models interact with external resources.

Problem MCP Solves:

  • AI models need access to real-time data
  • Different tools have different APIs
  • Security and authentication concerns
  • Need for standardized integration

MCP Solution:

  • Standardized protocol for tool access
  • Secure, authenticated connections
  • Easy integration with any AI model
  • Extensible and flexible

MCP Architecture

MCP Components

1

MCP Client

My application (FastAPI backend) that wants to use MCP tools

2

MCP Server

External service (GitHub MCP, Playwright MCP) that provides tools and resources

3

Transport Layer

Communication method (stdio, HTTP, SSE) - I use stdio (standard input/output)

4

Tools & Resources

Functions and data that MCP servers expose (e.g., list_commits, take_screenshot)

7 / 15

MCP Communication Flow

// 1. Client discovers available tools
await mcpClient.list_tools()
// 2. Client calls a tool with parameters
await mcpClient.call_tool(
"list_commits",
{ "owner": "vercel", "repo": "next.js" }
)
// 3. Server executes and returns result
// Returns: JSON with commit data

üîë Key Points

Stdio Transport: MCP servers run as separate processes, communicating via standard input/output streams. This allows secure, isolated execution.

JSON-RPC Protocol: All communication uses JSON-RPC 2.0 format for structured requests and responses.

MCP Servers in Our Project

GitHub MCP

Purpose: Fetch GitHub repository data

Tools: list_commits, list_issues, list_pull_requests, get_repository

Playwright MCP

Purpose: Browser automation and screenshots

Tools: browser_navigate, browser_take_screenshot

Example: Capture screenshot of GitHub repository page

üîß How They Work Together

When a user asks "Analyze vercel/next.js", GPT-4o-mini decides to call both servers: GitHub MCP for data and Playwright MCP for visual capture. Our backend orchestrates these calls in parallel for efficiency.
